# Suche

Personal search portal and start page. Six integrated search engines, per-user link buttons, and per-user RSS feeds. Runs at `http://localhost/suche.test` (dev) and `https://suche.eriks.cloud` (prod/akadbrain).

## Features

- **Search engines** — Google, Wikipedia, Geizhals, Amazon, Pons, Adobe Stock. Configured in `inc/search_engines.yaml`. Keyboard shortcut per engine (Alt+G, Alt+W, …).
- **Link buttons** — per-user grid of customisable links. Eight colour variants, optional icon or image. Drag-to-reorder.
- **RSS feed reader** — per-user feed list with card grid. 10-minute disk cache with stale-while-error fallback.
- **Authentication** — password login + TOTP 2FA via `erikr/auth`. Password reset by email.
- **Admin console** — user management and audit log (same as the rest of the ecosystem).

## Tech stack

PHP 8.2+ · MariaDB · Vanilla JS (no build step) · `erikr/auth` (Composer path dep) · `symfony/yaml`

Shared CSS library (`~/Git/css_library`) symlinked at `web/css/shared/`. No Bootstrap, no CDN assets.

## Quick start

### 1. Config

```bash
python3 ../mcp/generate.py --app suche --target local
```

Creates `config.yaml` with DB credentials, SMTP, base URL. See `config.example.yaml` for the key structure.

### 2. Dependencies

```bash
composer install
```

### 3. Migrations

```bash
php scripts/migrate.php
```

Idempotent — safe to run on every deploy. Creates `s_db_migrations`, `s_buttons`, `s_feeds` and seeds default data.

### 4. Run

Point any PHP-capable server at `web/`:

```bash
php -S localhost:8000 web/
```

Or via the Apache alias configured at `/etc/apache2/other/suche.conf` (dev):

```
http://localhost/suche.test/
```

## Deployment

```bash
# Generate config for target (local | akadbrain | world4you)
python3 ../mcp/generate.py --app suche --target akadbrain

# Deploy (rsync + remote migrations)
python3 ../mcp/deploy.py suche akadbrain
```

The deploy rsyncs `web/`, `inc/`, and `vendor/` to the target. `data/cache/rss/` is excluded from rsync and created on the server with 777 permissions on first deploy.

## DB tables

All three tables live in the shared `auth` database with the `s_` prefix:

| Table | Purpose |
|---|---|
| `s_db_migrations` | Tracks applied migrations |
| `s_buttons` | Per-user link buttons |
| `s_feeds` | Per-user RSS feeds |

## Tests

```bash
vendor/bin/phpunit
```

Unit tests for button validation and search engine config parsing. No DB required.
