## 1. Architecture

Suche is a PHP app with no build step, no front-end framework, and no server-side template engine. Pages are plain PHP files that output HTML directly after calling `render_header()`. JS is vanilla ES6 in `web/js/app.js`.

### Two DB connections per request

| Variable | Driver | DB | Used for |
|---|---|---|---|
| `$con` | MySQLi | `jardyx_auth` | auth operations (auth_accounts, auth_log) |
| `$pdo` | PDO | `jardyx_auth` | app data (s_buttons, s_feeds) |

Both connections point at the same database. The separation exists because `erikr/auth` expects a MySQLi handle while all app queries use PDO prepared statements.

### Request lifecycle

```
web/index.php
  require_once inc/initialize.php     ← config, $con, $pdo, session, CSP nonce
  auth_require()                      ← redirect to login.php if not authenticated
  $uid = $_SESSION['id']
  render_header('Start', 'home')
  … page-specific PHP …
  render_footer()
```

Every authenticated page follows this pattern. JSON endpoints (`api/buttons.php`, `api/feeds.php`) skip `render_header/footer` and return `Content-Type: application/json`.

## 2. Configuration

Config comes from `config.yaml` (gitignored, generated by `mcp/generate.py`). The canonical template is `config.example.yaml`.

### Structure

```yaml
target: local                    # local | akadbrain | world4you

db:                              # app PDO connection
  host: localhost
  socket: /tmp/mysql.sock        # optional; preferred over host when set
  name: jardyx_auth
  user: suche
  password: …

auth_db:                         # MySQLi connection for erikr/auth
  host: localhost
  socket: /tmp/mysql.sock
  name: jardyx_auth
  user: suche
  password: …

smtp:
  host: smtp.world4you.com
  port: 587
  user: catchall@jardyx.com
  password: …
  from: suche@jardyx.com
  from_name: Suche

app:
  base_url: http://localhost/suche.test   # no trailing slash
  env: dev                               # dev | prod
```

### Constants defined in `inc/initialize.php`

| Constant | Source |
|---|---|
| `APP_NAME` | `'Suche'` |
| `APP_VERSION` | `'3.0.0'` |
| `APP_BUILD` | date string e.g. `'2026-04-15'` |
| `APP_ENV` | `config.app.env` |
| `APP_BASE_URL` | `config.app.base_url` (trailing slash stripped) |
| `APP_SLUG` | `'suche'` — used by `appendLog()` as the `origin` field |

| `SMTP_HOST/PORT/USER/PASS/FROM/FROM_NAME` | from smtp config |
| `RATE_LIMIT_FILE` | `__DIR__ . '/../data/ratelimit.json'` |

## 3. Bootstrap chain

```
web/index.php
  └── require_once inc/initialize.php
        ├── require_once vendor/autoload.php
        ├── require_once inc/config.php          (suche_load_config)
        ├── define constants
        ├── new mysqli(...)                       → $con
        ├── new PDO(...)                          → $pdo
        └── auth_bootstrap([...])                 (session + CSP + CSRF)
```

Pages that only need auth (login, avatar, logout) may include `inc/initialize.php` directly without opening the PDO connection.

### CSP nonce

`auth_bootstrap()` generates `$_cspNonce` (a per-request nonce). All inline `<script>` blocks must include `nonce="<?= $_cspNonce ?>"`. The layout helpers (`render_header`, `render_footer`) and `api.php` inline scripts use it. CSP `style-src` allows `self` only — no external CSS.

## 4. Database schema

All tables are in the shared `jardyx_auth` database. The `s_` prefix namespaces them away from the auth tables.

### `s_db_migrations`

Tracks which SQL migration files have been applied.

```sql
CREATE TABLE s_db_migrations (
  id         INT UNSIGNED NOT NULL AUTO_INCREMENT,
  filename   VARCHAR(128) NOT NULL UNIQUE,
  applied_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (id)
)
```

### `s_buttons`

Per-user link buttons displayed on the main page.

```sql
CREATE TABLE s_buttons (
  id         INT UNSIGNED NOT NULL AUTO_INCREMENT,
  user_id    INT(11) NOT NULL,
  caption    VARCHAR(64) NOT NULL,
  url        TEXT NOT NULL,                        -- must begin with http(s)://
  target     ENUM('_blank','_self') DEFAULT '_blank',
  variant    VARCHAR(32) DEFAULT 'btn-default',    -- see variants below
  icon       VARCHAR(64) DEFAULT NULL,             -- emoji or class string
  img_url    VARCHAR(255) DEFAULT NULL,            -- overrides icon when set
  sort       INT DEFAULT 100,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  KEY idx_user_sort (user_id, sort),
  CONSTRAINT fk_s_buttons_user
    FOREIGN KEY (user_id) REFERENCES auth_accounts(id) ON DELETE CASCADE
)
```

**Allowed variants** (validated server-side in `buttons_validate()`):
`btn-default`, `btn-primary`, `btn-success`, `btn-warning`, `btn-danger`, `btn-secondary`, `btn-dark`, `btn-light`

### `s_feeds`

Per-user RSS feed subscriptions.

```sql
CREATE TABLE s_feeds (
  id         INT UNSIGNED NOT NULL AUTO_INCREMENT,
  user_id    INT(11) NOT NULL,
  title      VARCHAR(64) NOT NULL,
  url        TEXT NOT NULL,
  sort       INT DEFAULT 100,
  enabled    TINYINT(1) DEFAULT 1,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  KEY idx_user_sort (user_id, sort),
  CONSTRAINT fk_s_feeds_user
    FOREIGN KEY (user_id) REFERENCES auth_accounts(id) ON DELETE CASCADE
)
```

### Seed data

Migration `004_seed_s_buttons.sql` inserts 13 buttons for user `erikr` and 4 for `armin`, mapped from the 2015 XML colour scheme. Migration `005_seed_s_feeds.sql` seeds 4 feeds per user: MacTechNews.de, Der Standard, Standard/Web, ORF Science.

## 5. Search engines

Configured statically in `inc/search_engines.yaml`. Not user-editable. Loaded once per request by `search_engines_load()` (cached in a static variable).

### YAML structure per engine

```yaml
engines:
  google:
    label: Google
    action: https://www.google.com/search
    method: GET
    accesskey: g
    fields:
      - name: q
        type: text
        placeholder: Google …
    hidden:                        # extra hidden inputs added to the form
      - name: hl
        value: de
    site_restrict:                 # optional — renders a <select> prepended to the input
      - label: Web
        value: ''
      - label: Maps
        value: maps.google.com
```

### `render_search_form(array $engine): void`

Outputs a `<form>` block. `method` defaults to `GET`. The `site_restrict` select sets a hidden `as_sitesearch` field via JavaScript. Accesskey binds Alt+{key} (or Ctrl+Alt on some OSes) to focus the input.

## 6. Buttons

### Model functions (`inc/buttons.php`)

| Function | Signature | Description |
|---|---|---|
| `buttons_for_user` | `(PDO $pdo, int $uid): array` | SELECT all buttons for user, ORDER BY sort ASC |
| `buttons_get` | `(PDO $pdo, int $uid, int $id): ?array` | Fetch single button (checks user_id ownership) |
| `buttons_create` | `(PDO $pdo, int $uid, array $data): int` | INSERT, returns new id |
| `buttons_update` | `(PDO $pdo, int $uid, int $id, array $data): bool` | UPDATE (WHERE id=? AND user_id=?) |
| `buttons_delete` | `(PDO $pdo, int $uid, int $id): bool` | DELETE (WHERE id=? AND user_id=?) |
| `buttons_reorder` | `(PDO $pdo, int $uid, array $ids): void` | UPDATE sort in a loop, transaction-wrapped |
| `buttons_validate` | `(array $data): array` | Returns `['ok' => bool, 'errors' => string[]]` |

`buttons_validate()` checks: caption ≤ 64 chars, url starts with `http://` or `https://`, target is `_blank` or `_self`, variant is one of the 8 allowed values, img_url is a valid URL or empty.

### Render helper

`render_button(array $b): void` — outputs one `<a class="btn btn-{variant}">` element. If `img_url` is set, wraps an `<img>` inside; otherwise outputs the icon string (if set) followed by caption.

## 7. RSS feeds

### Model functions (`inc/feeds.php`)

Same CRUD pattern as buttons: `feeds_for_user`, `feeds_get`, `feeds_create`, `feeds_update`, `feeds_delete`, `feeds_reorder`, `feeds_validate`.

`feeds_validate()` checks: title ≤ 64 chars, url starts with `http://` or `https://`.

### Fetch and render (`inc/rss.php`)

**`rss_fetch(string $url): ?SimpleXMLElement`**

1. Derive a cache path: `data/cache/rss/` + MD5 of URL + `.xml`
2. If cache file is ≤ 10 minutes old, load and parse it
3. Otherwise: fetch with `file_get_contents()` using a spoofed Chrome user-agent, 3-second timeout, `LIBXML_NONET` flag. On success, write to cache
4. On fetch failure, fall back to stale cache if it exists (stale-while-error)
5. Returns parsed `SimpleXMLElement` or `null` on complete failure

**`rss_render(SimpleXMLElement $xml, int $limit = 8): string`**

Extracts up to `$limit` `<item>` elements. For each, finds the first image either from an `<img>` tag inside `<description>` (regex) or from `<enclosure url type="image/*">`. Returns an HTML string of `<div class="rss-cards">` cards. Each card links to `<link>`, shows title and publication date.

## 8. Authentication & session

Handled entirely by `erikr/auth`. Suche does not implement its own auth logic.

### Session fields set on login

| Key | Content |
|---|---|
| `$_SESSION['loggedin']` | `true` |
| `$_SESSION['id']` | user ID (int) |
| `$_SESSION['username']` | username string |
| `$_SESSION['email']` | email |
| `$_SESSION['rights']` | `'Admin'` or `''` |
| `$_SESSION['debug']` | bool |

### Auth gates

- **`auth_require()`** — redirects to `login.php` if not logged in. Used on every page except login, authentication, avatar, forgotPassword, setpassword, executeReset, totp_verify.
- **`admin_require()`** — additionally checks `$_SESSION['rights'] === 'Admin'`. Used in `admin.php` and `api.php` admin endpoints.

### TOTP 2FA

If a user has 2FA enrolled, `auth_login()` does not set `$_SESSION['loggedin']`. Instead it sets `$_SESSION['auth_totp_pending']` and redirects to `web/totp_verify.php`, which calls `auth_totp_complete()` on POST.

### CSRF

Every state-changing POST form includes `<?= csrf_input() ?>` (a hidden `_csrf_token` field). Handlers call `csrf_verify()` before processing. JSON API endpoints also require valid CSRF — the JS reads the token from `document.cookie` or a meta tag and includes it as `X-CSRF-Token`.

### Audit log

Every significant action calls `appendLog($con, $context, $activity)`. The `origin` field defaults to `APP_SLUG` (`'suche'`). Entries are visible in the admin log tab.

## 9. HTTP API

Two sets of endpoints handle state-changing operations:

### Button and feed CRUD (`web/api/`)

Both `buttons.php` and `feeds.php` follow the same pattern:

```
POST /api/buttons.php?action={create|update|delete|reorder}
POST /api/feeds.php?action={create|update|delete|reorder}
```

All require: authenticated session + valid CSRF token (checked via `csrf_verify()`). Return `application/json`.

**Response shape:**

```json
{ "ok": true, "id": 42 }            // create
{ "ok": true }                       // update, delete, reorder
{ "ok": false, "error": "…" }        // any failure
```

**`create`** — validates input, inserts row, returns new `id`.  
**`update`** — validates input, verifies ownership (`WHERE user_id = :uid`), updates row.  
**`delete`** — verifies ownership, deletes row.  
**`reorder`** — accepts `ids[]` array, updates `sort` values in a transaction.

### Admin API (`web/api.php`)

```
POST /api.php?action={admin_user_create|admin_user_edit|admin_user_reset|admin_user_delete}
```

Requires Admin rights. CSRF-protected. Delegates to `admin_*_user()` helpers from `erikr/auth`.

## 10. Admin console

`web/admin.php` — three-tab layout per ecosystem convention:

| Tab | Content |
|---|---|
| **App** | (placeholder — no app-specific parameters yet) |
| **Users** | User table with Edit / Deactivate / Reset password / Revoke 2FA / Delete actions |
| **Log** | Read-only view of `auth_log`, filterable by app / context / user / date / keyword |

Tab switching is client-side only (no page reload). All write actions are POST + CSRF via `adminPost()` JS helper. User table reuses the shared `.table` component. Log tab paginates at 50 rows/page via the shared `.pagination` component.

## 11. Layout system

### `render_header(string $title, string $activeNav = ''): void`

Outputs `<!DOCTYPE html>` through the closing `</header>` tag. Includes:
- `<meta>` charset, viewport, CSP nonce meta
- Favicon set (6 icons from `web/assets/`)
- CSS `<link>` tags in order: `shared/theme.css`, `shared/reset.css`, `shared/layout.css`, `shared/components.css`, `css/app.css`
- `.app-header` with left cluster (logo + wordmark) and right cluster (nav + user dropdown)
- User dropdown: Preferences / Password & 2FA / Admin (if admin) / Help / Logout

### `render_footer(): void`

Outputs the `.app-footer` (three-column grid: Impressum / copyright / version) and closing `</body></html>`. Also outputs `<script src="js/app.js">`.

### `$base` variable

`inc/initialize.php` sets `$base = rtrim(dirname($_SERVER['SCRIPT_NAME']), '/')`. All URLs in templates and layout use `$base . '/path'` — never hardcoded paths. This makes the app relocatable under any subpath.

## 12. CSS architecture

Load order (hard requirement):

```html
<link rel="stylesheet" href="css/shared/theme.css">
<link rel="stylesheet" href="css/shared/reset.css">
<link rel="stylesheet" href="css/shared/layout.css">
<link rel="stylesheet" href="css/shared/components.css">
<link rel="stylesheet" href="css/app.css">
```

`web/css/shared/` is a symlink to `~/Git/css`. Deployed with `rsync --copy-links` so the symlink is resolved at the destination — no symlinks on the production server.

### Token conventions

All colours use `--color-*` custom properties from `shared/theme.css`. App-specific tokens may be added to `css/app.css` as `:root` overrides but must follow the `--color-*` naming. No raw hex values in `app.css`.

### Dark mode

Three-block pattern in `shared/theme.css`:

```css
:root { /* light defaults */ }
[data-theme="dark"] { /* explicit dark */ }
@media (prefers-color-scheme: dark) {
  :root:not([data-theme="light"]) { /* auto dark */ }
}
```

The theme toggle in the user dropdown writes `data-theme` to `<html>` and persists the preference to the server via `POST /api.php?action=set-theme`.

## 13. JavaScript

All JS lives in `web/js/app.js` (~90 lines, no dependencies).

### Capabilities

- **Tab switching** — `.tab-link` clicks add/remove `active` class on panels. Used in preferences and admin.
- **Theme switcher** — reads/writes `data-theme` on `<html>`, POSTs to `set-theme` endpoint.
- **`apiPost(action, data)`** — thin wrapper around `fetch()` that includes the CSRF token header, decodes JSON, surfaces errors as alerts.
- **Alert helper** — `showAlert(type, message)` prepends a dismissible alert to `#alerts`.
- **Drag-to-reorder** — native HTML5 drag events on button/feed rows in preferences. Calls `reorder` endpoint on drop.

### CSP compliance

All inline scripts in PHP files must include `nonce="<?= $_cspNonce ?>"`. `app.js` is an external file so it needs no nonce.

## 15. Security model

### CSRF

All state-changing forms: hidden `_csrf_token` input via `csrf_input()`. All state-changing fetch calls: `X-CSRF-Token` header. Every handler calls `csrf_verify()` before touching the DB; non-compliant requests get a 400 or redirect.

### SQL injection

All DB queries use PDO prepared statements with named or positional placeholders. No string interpolation into SQL. User-owned rows are always fetched with `WHERE id = :id AND user_id = :uid` to prevent horizontal privilege escalation.

### XSS

All user-generated content echoed to HTML passes through `htmlspecialchars($val, ENT_QUOTES, 'UTF-8')`. The shared `addAlert()` helper calls `htmlentities()` on the message before storing in session.

### XXE

RSS fetching uses `LIBXML_NONET` to prevent entity-based network requests.

### Content Security Policy

Set by `auth_bootstrap()`. Inline scripts require the per-request nonce. External resources are restricted to `self`. The only exception is potential CDN CSS for Flatpickr (if used); add to `style-src` in the `auth_bootstrap()` call.

### Rate limiting

`RATE_LIMIT_FILE` tracks failed login attempts. Implemented inside `erikr/auth` — suche does not manage this.

### Avatar MIME whitelist

`web/avatar.php` only serves MIME types in the allowed list (`image/jpeg`, `image/png`, `image/gif`, `image/webp`). DB-stored MIME values are never reflected into `Content-Type` without whitelist check.

### Logout

`web/logout.php` accepts POST only and requires a valid CSRF token. The logout link in the user dropdown is a `<form method="post">` with a submit button — never a plain `<a>` tag.
